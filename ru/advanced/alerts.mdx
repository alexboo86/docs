# Оповещения

Оповещения уведомляют вас при выполнении определённых рыночных условий, чтобы вам не нужно было постоянно наблюдать за графиком. Cluster Terminal поддерживает две категории оповещений: **ценовые оповещения на инструментах рисования** и **оповещения на основе индикаторов**. Оба типа воспроизводят звуковые уведомления и визуальные индикаторы при срабатывании.

{/* screenshot: alert configuration dialog showing condition and mode options */}

## Обзор

| Категория оповещений | Источник | Триггер |
|----------------------|----------|---------|
| **Ценовые оповещения** | Инструменты рисования (линии, лучи) | Цена пересекает, поднимается выше или опускается ниже нарисованного уровня |
| **Оповещения индикаторов** | Правила оповещений индикаторов | Специфические события индикаторов (сигналы, дивергенции, тесты зон) |

## Ценовые оповещения на инструментах рисования

Три инструмента рисования поддерживают ценовые оповещения:

| Инструмент рисования | Условия оповещения |
|----------------------|-------------------|
| **Горизонтальная линия** | Пересечение, Цена выше, Цена ниже |
| **Трендовая линия** | Пересечение, Цена выше, Цена ниже |
| **Луч** | Пересечение, Цена выше, Цена ниже |

### Настройка ценового оповещения

<Steps>
  <Step title="Нарисуйте линию">
    Выберите инструмент Горизонтальная линия (**H**), Трендовая линия (**T**) или Луч (**R**) и нарисуйте его на графике на целевом ценовом уровне.
  </Step>
  <Step title="Включите оповещение">
    Щёлкните правой кнопкой мыши по рисунку, чтобы открыть контекстное меню, и выберите **Включить оповещение** (или нажмите иконку колокольчика в панели свойств рисунка).
  </Step>
  <Step title="Настройте условия">
    Выберите условие оповещения:
    - **Пересечение** — срабатывает при пересечении ценой линии в любом направлении
    - **Выше** — срабатывает при движении цены выше линии
    - **Ниже** — срабатывает при движении цены ниже линии
  </Step>
  <Step title="Установите режим оповещения">
    - **Однократно** — срабатывает один раз, затем автоматически отключается
    - **Повторяющийся** — срабатывает каждый раз при переходе условия из ложного в истинное
  </Step>
  <Step title="Выберите звук">
    Выберите звук оповещения из доступных синтезированных тонов. Разные звуки помогают различать оповещения.
  </Step>
</Steps>

{/* screenshot: right-click menu on horizontal line showing alert condition options */}

### Описание условий оповещений

| Условие | Когда срабатывает |
|---------|-------------------|
| **Пересечение** | Цена закрытия последней свечи переходит с одной стороны линии на другую. Работает как для пересечений вверх, так и вниз. |
| **Цена выше** | Цена закрытия последней свечи находится выше уровня линии. В режиме «Однократно» срабатывает на первом баре, который закрывается выше. |
| **Цена ниже** | Цена закрытия последней свечи находится ниже уровня линии. В режиме «Однократно» срабатывает на первом баре, который закрывается ниже. |

Для **трендовых линий** и **лучей** уровень оповещения меняется со временем по мере продления линии. Цена сравнивается со значением линии на временной координате текущего бара, а не с фиксированной ценой.

### Режимы оповещений

| Режим | Поведение |
|-------|-----------|
| **Однократно** | Срабатывает один раз при первом выполнении условия, затем автоматически отключается. Иконка оповещения удаляется с рисунка. Полезно для одноразовых уведомлений о пробое. |
| **Повторяющийся** | Срабатывает каждый раз при переходе условия из ложного в истинное. Требует сброса (условие должно стать ложным) перед повторным срабатыванием. Полезно для боковых рынков, где цена многократно тестирует уровень. |

<Tip>
Размещайте горизонтальные линии на ключевых уровнях поддержки/сопротивления с оповещением «Пересечение», чтобы получать уведомления, когда цена достигает интересующих вас уровней, даже когда вы не за экраном.
</Tip>

## Оповещения индикаторов

18 индикаторов поддерживают встроенные оповещения. Каждый индикатор определяет специфические события, релевантные для его типа анализа. Оповещения индикаторов настраиваются через панель настроек индикатора.

### Индикаторы с поддержкой оповещений

| Индикатор | События оповещений |
|-----------|-------------------|
| **Bollinger Bands** | Цена касается верхней/нижней полосы, обнаружено сжатие полос |
| **RSI** | Вход в зону перекупленности/перепроданности, выход из зоны |
| **RSI Divergence** | Обнаружена новая бычья/медвежья дивергенция |
| **MACD** | Пересечение сигнальной линии вверх/вниз, разворот гистограммы, пересечение нулевой линии |
| **Session Levels** | Цена приближается к максимуму/минимуму/открытию сессии, тест уровня |
| **Absorption Alpha** | Новый сигнал абсорбции (обнаружена крупная скрытая покупка/продажа), тест линии |
| **Delta Divergence** | Обнаружена дивергенция цены и дельты (цена вверх, дельта вниз или наоборот) |
| **CVD Divergence** | Дивергенция кумулятивного объёма дельты с ценой |
| **Orderflows Divergence** | Сигнал дивергенции потока ордеров |
| **Stacked Imbalance** | Сформирована новая зона стакированного дисбаланса, тест линии |
| **Clouds** | Зона облака сформирована, приближение к зоне облака, пробой облака |
| **DOM on Chart** | Обнаружен крупный уровень ордеров, уровень поглощён |
| **Liquidation Levels** | Приближение к крупному кластеру ликвидаций, начало каскада ликвидаций |
| **FVG** | Сформирован новый Fair Value Gap, зона FVG заполнена |
| **Market Structure** | Break of structure (BOS) бычий/медвежий, Change of character (CHoCH) бычий/медвежий |
| **Order Blocks** | Сформирован новый блок ордеров, приближение к зоне, зона пробита |

### Включение оповещений индикаторов

<Steps>
  <Step title="Откройте настройки индикатора">
    Нажмите на название индикатора или значок шестерёнки в панели индикаторов, чтобы открыть его конфигурацию.
  </Step>
  <Step title="Перейдите к правилам оповещений">
    Прокрутите до раздела **Правила оповещений** в настройках индикатора. В этом разделе перечислены все доступные события оповещений для данного индикатора.
  </Step>
  <Step title="Включите нужные события">
    Активируйте события оповещений, которые хотите получать. Каждое событие имеет свой переключатель включения/выключения.
  </Step>
  <Step title="Выберите звук оповещения (опционально)">
    Выберите звук для оповещений этого индикатора. Использование разных звуков для разных индикаторов помогает определить источник на слух.
  </Step>
  <Step title="Сохраните настройки">
    Закройте панель настроек. Оповещения активны немедленно.
  </Step>
</Steps>

{/* screenshot: indicator settings panel with Alert Rules section expanded showing event toggles */}

<Note>
Не все индикаторы поддерживают оповещения. В таблице выше перечислены все 18 индикаторов с возможностью оповещений. Для индикаторов, не указанных в списке, вы можете использовать ценовые оповещения на инструментах рисования в качестве альтернативы.
</Note>

### Типы событий оповещений

Разные индикаторы используют разные типы событий в зависимости от того, что они измеряют:

| Тип события | Описание |
|-------------|----------|
| `new_signal` | Индикатором сгенерирован новый аналитический сигнал |
| `new_divergence` | Обнаружена дивергенция между ценой и индикатором |
| `line_tested` | Цена коснулась или приблизилась к значимому уровню индикатора |
| `zone_approached` | Цена вошла в зону, определённую индикатором (например, зона блока ордеров) |
| `zone_broken` | Цена пробила ранее определённую зону |
| `level_absorbed` | Значимый уровень был поглощён (ордера исполнены, а не сняты) |

## Поведение оповещений

При срабатывании оповещения одновременно происходят два события:

- **Звук**: воспроизводится синтезированное звуковое уведомление через системный аудиовыход
- **Визуальное**: вызвавший срабатывание элемент подсвечивается на графике и/или появляется значок в панели инструментов

Звуки оповещений генерируются внутренне с помощью Web Audio API — внешние звуковые файлы не требуются. Доступно несколько типов звуков с различными тональными характеристиками (пинг, перезвон, сигнал тревоги и т.д.).

### Настройка звука

| Параметр | Описание |
|----------|----------|
| **Тип звука** | Выбор из встроенных синтезированных тонов (настраивается для каждого оповещения) |
| **Общая громкость** | Глобальная регулировка громкости для всех оповещений (в Настройки > Звук) |
| **Включение/отключение** | Глобальный переключатель звука (в Настройки > Звук) |

<Warning>
Убедитесь, что системный звук не отключён, если вы полагаетесь на звуковые оповещения. Терминал не может обойти системное отключение звука. Протестируйте оповещения после настройки, чтобы убедиться, что вы слышите уведомления.
</Warning>

### История оповещений

Сработавшие оповещения логируются с меткой времени `alertTriggeredAt`. Это позволяет просматривать, когда оповещения срабатывали, даже если вы не наблюдали за экраном в тот момент.

## Комбинирование оповещений для высоковероятных сетапов

Настоящая сила оповещений раскрывается при сочетании ценовых оповещений с оповещениями индикаторов:

### Уровень поддержки + Абсорбция

1. Нарисуйте горизонтальную линию на ключевом уровне поддержки
2. Установите оповещение «Цена ниже» (однократно) на линии
3. Включите оповещения `new_signal` на Absorption Alpha
4. Когда оба оповещения срабатывают примерно одновременно, у вас тест поддержки с подтверждённой абсорбцией — высоковероятный сетап на лонг

### Пробой трендовой линии + Структура рынка

1. Нарисуйте восходящую трендовую линию под бычьей структурой
2. Установите оповещение «Пересечение» на трендовой линии
3. Включите оповещения BOS/CHoCH на Market Structure
4. Одновременный пробой трендовой линии и сигнал BOS подтверждают смену тренда

### Bollinger Bands + Дивергенция RSI

1. Включите оповещение «Цена касается нижней полосы» на Bollinger Bands
2. Включите оповещение «Новая бычья дивергенция» на RSI Divergence
3. Касание нижней полосы при бычьей дивергенции RSI сигнализирует о сетапе на возврат к среднему

<Tip>
Начните с небольшого количества оповещений и добавляйте больше по мере того, как поймёте, какие сигналы наиболее релевантны для вашей стратегии. Слишком много одновременных оповещений создают шум, на который сложнее реагировать, чем на несколько хорошо выбранных.
</Tip>

## Ограничения

| Ограничение | Подробности |
|-------------|-------------|
| **Требуется запущенный терминал** | Оповещения срабатывают только при работающем приложении |
| **Без push-уведомлений** | Оповещения локальны для терминала (только звук + визуал) |
| **Без email или SMS** | Внешние каналы уведомлений в настоящее время не поддерживаются |
| **Оценка по закрытию свечи** | Ценовые условия оцениваются при закрытии каждой новой свечи, а не тик за тиком |
| **Привязка к рабочему пространству** | Оповещения привязаны к рабочему пространству — они срабатывают только для символов активного рабочего пространства |

<Note>
Поскольку оповещения оцениваются по закрытию свечи (а не на каждом тике), очень быстрые ценовые всплески, которые пересекают уровень и возвращаются в пределах одной свечи, могут сработать или не сработать в зависимости от того, где свеча закроется. Для тиковой точности отслеживайте панель DOM напрямую.
</Note>
