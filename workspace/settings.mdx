# Настройки

Панель настроек позволяет настроить все аспекты Cluster Terminal — от подключения к серверу и визуальной темы до значений графиков по умолчанию, поведения загрузки данных и звуковых предпочтений.

Откройте Настройки через значок шестерёнки в заголовке или нажмите `Ctrl + ,`.

{/* screenshot: settings panel overview */}

## Подключение к серверу

URL сервера определяет, откуда Cluster Terminal получает все рыночные данные (свечи, кластеры, открытый интерес) и подключается для потоковой передачи данных в реальном времени через WebSocket.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **URL сервера** | Базовый URL сервера данных Cluster Terminal | Устанавливается при первом запуске |

<Steps>
  <Step title="Откройте Настройки">
    Нажмите значок шестерёнки или нажмите `Ctrl + ,`.
  </Step>
  <Step title="Измените URL сервера">
    В разделе **Сервер** введите полный URL, включая протокол и порт (например, `http://95.216.29.184:8080`).
  </Step>
  <Step title="Проверьте подключение">
    Приложение автоматически тестирует подключение при сохранении. Зелёная галочка означает успех; красный значок означает, что сервер недоступен.
  </Step>
</Steps>

<Warning>
  Изменение URL сервера отключает все активные WebSocket-потоки и перезагружает данные графиков. Изменяйте этот параметр только при миграции на другой экземпляр сервера.
</Warning>

## Внешний вид

### Язык

Cluster Terminal поддерживает несколько языков интерфейса.

| Язык | Статус |
|------|--------|
| **Английский** | Полная поддержка |
| **Русский** | Полная поддержка |

Смените язык в **Настройки > Внешний вид > Язык**. Изменение вступает в силу немедленно без перезапуска.

### Тема

| Тема | Описание |
|------|----------|
| **Тёмная** | Тёмный фон, светлый текст. Оптимизирована для длительной работы за экраном. |
| **Светлая** | Светлый фон, тёмный текст. |

{/* screenshot: dark vs light theme comparison */}

<Tip>
  Тёмная тема рекомендуется для трейдинга, чтобы снизить нагрузку на глаза при длительных сессиях, особенно при использовании нескольких мониторов.
</Tip>

## Настройки графика по умолчанию

Эти настройки управляют начальной конфигурацией при создании новой вкладки графика.

| Параметр | Описание | Варианты |
|----------|----------|----------|
| **Тип графика по умолчанию** | Тип графика для новых вкладок | Свечной, Кластерный, TPO, Тепловая карта |
| **Таймфрейм по умолчанию** | Начальный таймфрейм | 1m, 5m, 15m, 1h, 4h, 1D, 1W, 1M |
| **Символ по умолчанию** | Начальный символ | Любой доступный символ (например, BTCUSDT) |
| **Цвета свечей** | Цвета бычьих / медвежьих свечей | Палитра цветов |

<Note>
  Эти значения по умолчанию применяются только к новым вкладкам. Существующие вкладки сохраняют свои собственные настройки.
</Note>

## Загрузка данных

Управление загрузкой и кэшированием исторических данных.

| Параметр | Описание |
|----------|----------|
| **Прогрессивная загрузка** | Загрузка данных объёмов/кластеров частями с индикатором прогресса. Сохраняет отзывчивость интерфейса для больших диапазонов дат. |
| **Глубина истории** | Насколько далеко в прошлое загружать данные при открытии графика. Более глубокая история означает более длительную начальную загрузку. |

### Прогрессивная загрузка

При включении большие запросы данных (особенно кластерные объёмы) разбиваются на дневные порции и загружаются последовательно. Индикатор прогресса на панели инструментов графика показывает статус загрузки.

<Tip>
  Оставляйте прогрессивную загрузку включённой. Она предотвращает зависание интерфейса при загрузке месяцев кластерных данных, особенно для символов с высоким объёмом, таких как BTCUSDT.
</Tip>

## Настройки звука

Настройка звуковых уведомлений для оповещений и системных событий.

| Параметр | Описание |
|----------|----------|
| **Звуки оповещений** | Включение или отключение звука при срабатывании ценовых/индикаторных оповещений |
| **Громкость звука** | Общая громкость для всех звуков уведомлений |

Звуки оповещений синтезируются в браузере — внешние аудиофайлы не требуются. Каждый тип оповещения имеет свой уникальный звуковой паттерн, позволяющий различать ценовые оповещения, сигналы индикаторов и системные уведомления, не глядя на экран.

## Лицензия

Просмотр и управление статусом лицензии.

| Поле | Описание |
|------|----------|
| **Лицензионный ключ** | Ваш активный лицензионный ключ (формат CT-XXXX-XXXX-XXXX-XXXX) |
| **Статус** | Активна, Пробная или Истекла |
| **Функции** | Включённые функции (например, cloud_sync, priority_support) |
| **Устройства** | Количество активированных устройств из максимально допустимого |
| **Срок действия** | Дата истечения лицензии |

### Пробный режим

Если вы не ввели лицензионный ключ, Cluster Terminal работает в пробном режиме в течение 7 дней с полным функционалом. После истечения пробного периода для продолжения использования приложения необходим лицензионный ключ.

<Note>
  Длительность пробного периода может варьироваться. Администратор сервера может настроить пробный период через панель администратора.
</Note>

### Активация лицензии

<Steps>
  <Step title="Получите лицензионный ключ">
    Приобретите лицензию или получите её от администратора проекта. Ключи имеют формат `CT-XXXX-XXXX-XXXX-XXXX`.
  </Step>
  <Step title="Введите ключ">
    Перейдите в **Настройки > Лицензия** и вставьте ключ в поле лицензии.
  </Step>
  <Step title="Активируйте">
    Нажмите **Активировать**. Приложение проверит ключ на сервере и зарегистрирует ваше устройство.
  </Step>
</Steps>

<Warning>
  Каждая лицензия имеет максимальное количество устройств. Если вы достигли лимита, деактивируйте существующее устройство через панель администратора сервера или обратитесь в поддержку.
</Warning>

## Сброс настроек

Для сброса всех настроек к значениям по умолчанию очистите каталог данных приложения. В Windows он расположен по пути:

```
%APPDATA%/com.cluster-terminal/
```

<Warning>
  Сброс данных приложения удаляет все локальные рабочие пространства, активацию лицензии, состояние пробного периода и кэшированные данные. Это действие нельзя отменить.
</Warning>
