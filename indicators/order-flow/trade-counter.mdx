# Trade Counter

Trade Counter отображает **количество индивидуальных исполнений сделок на каждой свече**, разделённых на покупки и продажи. В то время как большинство индикаторов фокусируются на объёме (суммарном размере сделок), Trade Counter фокусируется на частоте сделок — сколько раз рынок совершил транзакцию, независимо от размера каждой из них.

Это различие критически важно: одна рыночная заявка на 100 BTC выглядит совершенно иначе, чем 10 000 заявок по 0.01 BTC, хотя суммарный объём идентичен. Trade Counter выявляет эту разницу.

## Как это работает

Для каждой свечи Trade Counter считывает данные кластера и подсчитывает:

- **Покупки**: количество индивидуальных исполнений по цене ask (агрессивные покупки)
- **Продажи**: количество индивидуальных исполнений по цене bid (агрессивные продажи)

Эти значения отображаются в виде подграфика, где покупки и продажи показаны как отдельные серии. Индикатор не агрегирует и не сглаживает данные — каждый бар представляет необработанный подсчёт за временной период свечи.

## Тип индикатора

**Подграфик** — отображается в отдельной панели под основным графиком цены со своей собственной осью Y.

## Настройки

### Цвета

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `buyColor` | color | `#22c55e` | Цвет для количества покупок |
| `sellColor` | color | `#ef4444` | Цвет для количества продаж |

### Режим отображения

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `displayMode` | select | `columns` | Стиль отрисовки: `line`, `columns` или `candles` |

- **`columns`** (по умолчанию): вертикальные столбцы от нуля. Покупки и продажи отрисовываются рядом. Наиболее распространённый выбор для сравнения участия.
- **`line`**: непрерывная линия, соединяющая значения количества сделок. Полезно, когда вы хотите видеть тренд частоты торговли во времени.
- **`candles`**: OHLC-представление количества сделок. Менее распространённый вариант, но полезен для выявления внутрибарных паттернов количества сделок на старших таймфреймах.

### Выделение аномалий

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `highlightAnomalies` | boolean | `false` | Включить статистическое обнаружение аномалий |
| `anomalyThreshold` | number | `3.0` | Количество стандартных отклонений выше среднего для квалификации как аномалия |
| `anomalyPeriod` | number | `50` | Окно обзора (бары) для расчёта среднего/стандартного отклонения |

При включении бары, где количество сделок превышает `среднее + (порог x стандартное отклонение)`, выделяются отличительным визуальным маркером. Порог 3.0 отмечает только экстремальные всплески — примерно верхние 0.1% баров.

### Градиентная интенсивность

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `gradientIntensity` | boolean | `false` | Масштабирование прозрачности бара по величине значения |
| `minOpacity` | number | `0.25` | Минимальная прозрачность для баров с наименьшими значениями |

При включении бары с большим количеством сделок выглядят более непрозрачными, а тихие бары становятся более прозрачными до минимального значения. Это создаёт эффект тепловой карты, который делает всплески мгновенно заметными без включения полного обнаружения аномалий.

### Ширина бара

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `barWidth` | number | `0.7` | Ширина столбца относительно расстояния между свечами (0.0–1.0) |

## Интерпретация Trade Counter

### Матрица: объём vs. количество сделок

Наиболее мощное применение Trade Counter — сравнение с суммарным объёмом. Это даёт четыре сценария:

| Объём | Количество сделок | Интерпретация |
|-------|-------------------|---------------|
| Высокий | Высокое | Широкое рыночное участие — новостные события, каскады ликвидаций |
| Высокий | Низкое | Мало крупных ордеров — институциональная / китовая активность |
| Низкий | Высокое | Много мелких ордеров — розничный шум, алгоритмический маркетмейкинг |
| Низкий | Низкое | Тихий рынок — низкий интерес, консолидация |

Комбинация "Высокий объём + Низкое количество сделок" является сигнатурой институциональной активности и основной причиной использования этого индикатора.

<Tip>
Сравнивайте Trade Counter с объёмом — расхождение между количеством сделок и объёмом выявляет институциональную активность. Когда объём растёт, но количество сделок остаётся на месте, крупный игрок, вероятно, исполняет блочные ордера.
</Tip>

### Обнаружение алгоритмов

Алгоритмы высокочастотной торговли генерируют характерный паттерн: чрезвычайно высокое количество сделок с индивидуально малыми размерами. Если Trade Counter показывает внезапный всплеск частоты, который не сопровождается пропорциональным всплеском объёма, вероятна алгоритмическая активность.

### Кульминационное истощение

На рыночных вершинах и дне часто наблюдается кульминационный всплеск как объёма, так и количества сделок, за которым следует резкое снижение количества сделок на следующих нескольких барах. Этот паттерн "истощения" указывает на то, что движение исчерпало участников, готовых продолжать в этом направлении.

## Сочетание с другими индикаторами

Trade Counter лучше всего работает вместе с:

- **Bar Statistics** — предоставляет дополнительную разбивку по объёму. Trade Counter даёт частоту; Bar Statistics даёт величину.
- **Delta** — когда дельта близка к нулю, но количество сделок высокое, обе стороны активно борются за контроль.
- **Absorption** — события абсорбции (цена остаётся на месте при высоком объёме) становятся более значимыми, когда количество сделок также повышено.
- **CVD** — дивергенция кумулятивной дельты в сочетании с аномалиями количества сделок может сигнализировать об истощении до разворота цены.

## Алерты

Trade Counter не поддерживает выделенные алерты. Для получения уведомлений о всплесках количества сделок включите `highlightAnomalies` и используйте визуальный подход — выделенные бары будут выделяться на графике.

Для автоматического оповещения об экстремальных значениях, связанных с объёмом, рассмотрите использование индикаторов Delta или Volume, которые имеют встроенные правила алертов.

## Замечания по производительности

Trade Counter — лёгкий индикатор. Он считывает данные о количестве сделок напрямую из предварительно рассчитанных кластерных блобов без дополнительной агрегации или преобразования. Включение выделения аномалий добавляет небольшой расчёт для каждого бара (скользящее среднее и стандартное отклонение), который незначителен даже на графиках с 10 000+ баров.

## Итого

Trade Counter отвечает на вопрос, на который объём сам по себе ответить не может: сколько индивидуальных участников внесли вклад в активность на каждом баре. Сравнивая частоту сделок с объёмом, вы можете отличить институциональные блочные ордера от розничного шума, обнаружить алгоритмическую активность и выявить кульминационное истощение в точках разворота.
