# Обзор индикаторов

Cluster Terminal предоставляет полный набор из **38 технических индикаторов**, охватывающих классический анализ, метрики потока ордеров, профили объёма и концепции умных денег. Каждый индикатор полностью настраиваем и может свободно комбинироваться с другими на любом графике.

## Открытие панели индикаторов

Чтобы добавить индикаторы на график:

1. Нажмите кнопку **Индикаторы** на панели инструментов графика (иконка колбы) или используйте горячую клавишу **`I`**.
2. Откроется браузер индикаторов, организованный по категориям: Классические, Поток ордеров, Метрики, Профили и SMC.
3. Используйте строку поиска вверху для фильтрации по имени.
4. Нажмите на любой индикатор, чтобы добавить его на активный график.

<Tip>
Вы можете добавить один и тот же индикатор несколько раз с разными настройками — например, три наложения SMA с периодами 20, 50 и 200.
</Tip>

## Типы индикаторов

Каждый индикатор относится к одному из трёх типов отображения, определяющих, где он отрисовывается на графике:

| Тип | Описание | Примеры |
|-----|----------|---------|
| **Overlay** | Отрисовывается прямо на ценовом графике, разделяя ось цен | SMA, EMA, VWAP, Bollinger Bands, Session Levels |
| **Subchart** | Отрисовывается на отдельной панели под основным графиком с собственной осью Y | RSI, MACD, CVD, Delta, Funding Rate |
| **Side Panel** | Отображается как вертикальная панель рядом с графиком | Volume Profile, TPO Profile |

Overlay-индикаторы не влияют на шкалу цен, если их значения существенно не расходятся с ценой. Subchart-индикаторы автоматически выделяют вертикальное пространство, и их размер можно изменить перетаскиванием разделителя между панелями.

## Добавление и удаление индикаторов

### Добавление

Выберите любой индикатор в браузере, чтобы добавить его. Индикатор мгновенно появится на графике с настройками по умолчанию. Автоматически откроется диалог настроек, чтобы вы могли изменить параметры перед закрытием.

### Удаление

Чтобы удалить индикатор:

1. Щёлкните правой кнопкой мыши на метке индикатора в легенде графика (overlay) или заголовке подграфика (subchart-индикаторы).
2. Выберите **Удалить** в контекстном меню.

Также можно открыть диалог настроек индикатора и нажать кнопку **Удалить** внизу.

### Изменение порядка

Subchart-индикаторы можно переупорядочить, перетаскивая их заголовки вверх или вниз. Порядок сохраняется в вашем рабочем пространстве.

## Общие настройки

Многие индикаторы разделяют стандартный набор параметров конфигурации. Они описаны здесь один раз, чтобы избежать повторений на страницах отдельных индикаторов.

### Режим отображения

```
displayMode: "line" | "columns" | "candles"
```

Управляет способом отрисовки серии данных индикатора:

| Режим | Описание |
|-------|----------|
| `line` | Непрерывная линия, соединяющая точки данных. Лучше всего для плавных осцилляторов (RSI, линия MACD). |
| `columns` | Вертикальные столбцы, привязанные к нулевой линии. Лучше всего для гистограммных данных (Delta, Volume). |
| `candles` | Тела свечей в стиле OHLC. Используется, когда индикатор выдаёт значения open/high/low/close. |

Не все режимы доступны для каждого индикатора — диалог настроек показывает только применимые варианты.

### Подсветка аномалий

При включении индикатор выделяет бары, являющиеся статистическими выбросами — значения, значительно превышающие недавнее среднее.

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `highlightAnomalies` | boolean | `false` | Включить/выключить подсветку аномалий |
| `anomalyThreshold` | number | 2.0–3.0 | Количество стандартных отклонений выше среднего для квалификации как аномалия |
| `anomalyPeriod` | number | 20–50 | Окно обратного просмотра (в барах) для расчёта среднего и стандартного отклонения |

Расчёт прост: для каждого бара вычисляются среднее и стандартное отклонение значения индикатора за последние `anomalyPeriod` баров. Если текущее значение превышает `среднее + (anomalyThreshold × stddev)`, оно отмечается как аномалия и отрисовывается с отличительным цветом подсветки.

<Tip>
Начните с порога по умолчанию и корректируйте в зависимости от количества подсветок. Более низкие значения (1.5–2.0) дают больше подсветок; более высокие (3.0+) отмечают только экстремальные выбросы.
</Tip>

### Градиентная интенсивность

При включении прозрачность баров масштабируется пропорционально величине значения относительно видимого диапазона.

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `gradientIntensity` | boolean | `false` | Включить/выключить градиентное масштабирование прозрачности |
| `minOpacity` | number | 0.25 | Минимальная прозрачность для баров с наименьшими значениями (диапазон 0.0–1.0) |

Бары с более высокими значениями выглядят более непрозрачными, а бары с низкими значениями затухают до `minOpacity`. Это обеспечивает интуитивный визуальный тепловой эффект без добавления дополнительных индикаторов.

### Ширина бара

```
barWidth: 0.7
```

Управляет шириной столбцов относительно интервала между свечами. Значение `1.0` означает, что столбцы касаются друг друга без зазора; `0.5` означает, что каждый столбец занимает половину доступного пространства. Значение по умолчанию `0.7` обеспечивает сбалансированный вид.

## Премиум визуальные стили

Четыре премиум-стиля рендеринга доступны для определённых индикаторов (в настоящее время их поддерживают 4 индикатора):

| Стиль | Описание |
|-------|----------|
| **Flat** | Стандартная заливка сплошным цветом без эффектов. По умолчанию. |
| **Neon** | Яркие, насыщенные цвета с мягким внешним свечением, напоминающим неоновую вывеску. |
| **Gradient** | Цветовой градиент сверху вниз внутри каждого бара или линии. |
| **Chrome** | Металлический отражающий блеск с градиентами бликов и теней. |

Премиум-стили являются чисто косметическими и не влияют на вычисления индикатора.

### Эффект свечения

Индикаторы, поддерживающие премиум-стили, также предоставляют настройки свечения:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `enableGlow` | boolean | `false` | Включить эффект внешнего свечения вокруг индикатора |
| `glowIntensity` | number | 0.5 | Яркость свечения (0.0–1.0) |
| `glowLayers` | number | 2 | Количество наложенных слоёв свечения для более выраженного эффекта |

Более высокие значения `glowLayers` создают более рассеянное, атмосферное свечение, но могут немного увеличить нагрузку на рендеринг при больших графиках.

## Оповещения

**18 из 38** индикаторов поддерживают настраиваемые правила оповещений. Когда условия индикатора выполняются, Cluster Terminal срабатывает оповещение со звуком и визуальным уведомлением.

Индикаторы с поддержкой оповещений отображают раздел **Оповещения** в диалоге настроек, где вы можете:

- Включить или выключить срабатывание оповещения для данного экземпляра индикатора.
- Выбрать, какие условия запускают оповещение (например, пересечение RSI выше зоны перекупленности, пересечение сигнала MACD).
- Установить период охлаждения, чтобы избежать повторных оповещений от одного условия.

Подробности о настройке и управлении оповещениями см. на странице документации [Оповещения](/ru/advanced/alerts).

## Категории индикаторов

Cluster Terminal организует индикаторы в пять категорий:

| Категория | Количество | Описание |
|-----------|-----------|----------|
| **Классические** | ~12 | Традиционный технический анализ: скользящие средние, осцилляторы, полосы, уровни |
| **Поток ордеров** | ~10 | Анализ на уровне сделок: дельта, CVD, абсорбция, метрики футпринта |
| **Метрики** | ~8 | Производные рыночные метрики: ставка финансирования, открытый интерес, ликвидации |
| **Профили** | ~4 | Визуализации профилей объёма и времени |
| **SMC** | ~4 | Концепции умных денег: блоки ордеров, разрывы справедливой стоимости, структура |

Просматривайте каждую категорию в боковой панели, чтобы узнать об отдельных индикаторах, их настройках и практических советах по использованию.

<Tip>
Вы можете комбинировать индикаторы из разных категорий на одном графике. Популярная комбинация — VWAP + Volume Profile + Delta — сочетание классической, профильной и потоковой перспектив.
</Tip>

## Рекомендации по производительности

Все индикаторы вычисляются на стороне клиента с использованием WebGL-ускоренного рендеринга через SciChart. Тем не менее, несколько рекомендаций помогут сохранить плавность работы:

- **Ограничивайте количество подграфиков**: более 5–6 subchart-панелей на одной вкладке графика могут уменьшить вертикальное пространство для каждой панели. Рассмотрите разделение по вкладкам.
- **Аномалии + градиент**: включение `highlightAnomalies` и `gradientIntensity` одновременно на многих индикаторах добавляет вычисления для каждого бара. На графиках с 10 000+ барами это может вызвать небольшую задержку при начальной загрузке.
- **Слои свечения**: установка `glowLayers` выше 3 на нескольких индикаторах нагружает GPU. Используйте экономно на менее мощном оборудовании.
