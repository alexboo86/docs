# Часто задаваемые вопросы

Распространённые вопросы о Cluster Terminal, его функциях, поддерживаемых данных и доступности платформы.

## Биржи и данные

<AccordionGroup>
  <Accordion title="Какие биржи поддерживаются?">
    Cluster Terminal в настоящее время поддерживает **Binance Futures (бессрочные контракты USDT-M) и Spot-рынки**. Это охватывает все активно торгуемые USDT-маржинальные бессрочные фьючерсы и основные спотовые торговые пары на Binance.

    Поддержка дополнительных бирж (Bybit, OKX) запланирована в будущих релизах.
  </Accordion>

  <Accordion title="Какие типы данных доступны?">
    Доступны следующие типы данных:

    | Тип данных | Описание |
    |------------|----------|
    | **Свечи (OHLCV)** | Стандартные свечные данные с открытием, максимумом, минимумом, закрытием и объёмом |
    | **Кластеры / Footprint** | Данные объёма по ценовым уровням, агрегированные из отдельных сделок |
    | **Открытый интерес** | OHLC открытого интереса во времени |
    | **Книга ордеров (DOM)** | Глубина bid/ask в реальном времени (только живые данные) |
    | **Ликвидации** | События принудительной ликвидации (только живые данные) |
    | **Ставка финансирования** | Ставка финансирования бессрочных фьючерсов (только живые данные) |
    | **Соотношение лонг/шорт** | Агрегированное соотношение длинных и коротких позиций (только живые данные) |
  </Accordion>

  <Accordion title="На какую глубину доступна историческая информация?">
    Исторические данные доступны с **даты листинга** каждого символа на Binance. Для основных символов, таких как BTCUSDT, это означает данные с 2019 года. Более новые символы имеют данные начиная с момента их листинга.

    Сервер импортирует исторические данные из публичных архивов данных Binance с проверкой контрольной суммы SHA256 для обеспечения точности.
  </Accordion>

  <Accordion title="Как часто обновляются данные?">
    - **Данные в реальном времени** (свечи, сделки, DOM): передаются непрерывно через WebSocket с задержкой менее секунды.
    - **Исторические данные**: сервер выполняет ежедневный импорт в 03:00 UTC для заполнения данных за предыдущий день.
    - **Список символов**: обновляется каждые 6 часов для учёта новых и снятых с торгов символов.
  </Accordion>

  <Accordion title="Откуда поступают данные?">
    Все данные поступают напрямую с Binance. Исторические данные импортируются из публичного архива данных Binance (ZIP-файлы с проверкой контрольной суммы). Данные в реальном времени передаются через WebSocket API Binance. Сервер Cluster Terminal агрегирует и кэширует эти данные для быстрой доставки настольному клиенту.
  </Accordion>
</AccordionGroup>

## Графики и анализ

<AccordionGroup>
  <Accordion title="Какие таймфреймы доступны?">
    Cluster Terminal поддерживает следующие таймфреймы:

    | Таймфрейм | Код |
    |-----------|-----|
    | 1 минута | 1m |
    | 5 минут | 5m |
    | 15 минут | 15m |
    | 1 час | 1h |
    | 4 часа | 4h |
    | 1 день | 1D |
    | 1 неделя | 1W |
    | 1 месяц | 1M |

    Кластерные данные доступны на всех таймфреймах — сервер агрегирует данные на уровне сделок в запрашиваемый таймфрейм на лету.
  </Accordion>

  <Accordion title="Что такое кластерный (footprint) график?">
    Кластерный график (также называемый footprint-графиком) показывает объём, проторгованный на каждом ценовом уровне внутри свечи. Вместо того чтобы видеть только общий объём бара, вы видите, как этот объём распределён по ценам.

    Это позволяет увидеть:
    - Где произошла наибольшая торговая активность внутри свечи
    - Объём покупок и продаж (дельту) на каждом ценовом уровне
    - Паттерны абсорбции, где были исполнены крупные ордера
    - Дисбалансы между агрессивными покупателями и продавцами

    Кластерные графики — это ключевая функция Cluster Terminal и основа многих индикаторов потока ордеров.
  </Accordion>

  <Accordion title="Какие типы графиков доступны?">
    | Тип графика | Описание |
    |-------------|----------|
    | **Свечной** | Стандартные OHLCV свечи |
    | **Кластерный / Footprint** | Объём по ценовым уровням внутри каждой свечи |
    | **TPO (Time Price Opportunity)** | График в стиле Market Profile |
    | **Тепловая карта** | Цветовая карта объёмов по цене и времени |
  </Accordion>

  <Accordion title="Сколько индикаторов доступно?">
    Более 50 индикаторов в нескольких категориях:

    - **Классические**: RSI, MACD, Bollinger Bands, EMA, SMA, VWAP и другие
    - **Объёмные**: Volume Profile, Session Levels, OI
    - **Поток ордеров**: Absorption Alpha, CVD, Delta Divergence, Stacked Imbalances, Clouds, DOM on Chart, Liquidation Levels
    - **Структурные**: Fair Value Gaps (FVG), Market Structure (BOS/CHoCH), Order Blocks

    Большинство индикаторов поддерживают настраиваемые оповещения. См. [Оповещения](/ru/advanced/alerts) для полного списка.
  </Accordion>
</AccordionGroup>

## Платформа и установка

<AccordionGroup>
  <Accordion title="Какие операционные системы поддерживаются?">
    Cluster Terminal в настоящее время доступен для **Windows** (64-бит). Приложение построено на Tauri 2 и использует WebGL для рендеринга графиков.

    Сборки для macOS и Linux запланированы, но пока недоступны.
  </Accordion>

  <Accordion title="Есть ли мобильная версия?">
    Нет. Cluster Terminal — это исключительно настольное приложение. Сложные графики, многопанельные раскладки и рендеринг WebGL требуют настольной среды. Мобильная версия в настоящее время не планируется.
  </Accordion>

  <Accordion title="Каковы системные требования?">
    | Компонент | Минимум | Рекомендуется |
    |-----------|---------|---------------|
    | **ОС** | Windows 10 (64-бит) | Windows 11 |
    | **RAM** | 4 ГБ | 8 ГБ или более |
    | **GPU** | Любая с WebGL 2.0 | Дискретная видеокарта |
    | **Дисплей** | 1366x768 | 1920x1080 или выше |
    | **Интернет** | Стабильное соединение | Соединение с низкой задержкой |

    Дискретная видеокарта рекомендуется для плавного рендеринга с большим количеством индикаторов и открытых панелей.
  </Accordion>

  <Accordion title="Как установить Cluster Terminal?">
    1. Загрузите последний установщик (файл `.exe`) из официального источника.
    2. Запустите установщик — это мастер установки NSIS.
    3. Следуйте инструкциям для завершения установки.
    4. Запустите Cluster Terminal из меню «Пуск» или ярлыка на рабочем столе.
    5. При первом запуске введите URL сервера и при необходимости активируйте лицензионный ключ.
  </Accordion>
</AccordionGroup>

## Лицензия и пробный период

<AccordionGroup>
  <Accordion title="Есть ли бесплатный пробный период?">
    Да. При первом запуске без лицензионного ключа Cluster Terminal переходит в **7-дневный пробный режим** с полной функциональностью. Банковская карта или регистрация не требуются.

    После истечения пробного периода для продолжения использования приложения необходим лицензионный ключ.
  </Accordion>

  <Accordion title="Что включает лицензия?">
    Лицензия предоставляет:
    - Неограниченный доступ ко всем типам графиков и индикаторам
    - Потоковая передача данных в реальном времени
    - Полный доступ к историческим данным
    - Дополнительные функции в зависимости от уровня лицензии (например, `cloud_sync`, `priority_support`)
  </Accordion>

  <Accordion title="На скольких устройствах можно использовать?">
    Каждая лицензия имеет настраиваемый лимит устройств (устанавливается администратором). При активации лицензии на новом устройстве регистрируется отпечаток устройства. Если вы достигли лимита, деактивируйте существующее устройство или обратитесь в поддержку.
  </Accordion>

  <Accordion title="Как активировать лицензию?">
    Перейдите в **Настройки > Лицензия**, введите лицензионный ключ (формат: `CT-XXXX-XXXX-XXXX-XXXX`) и нажмите **Активировать**. Приложение проверит ключ на сервере и зарегистрирует ваше устройство. Подробнее см. [Настройки](/ru/workspace/settings).
  </Accordion>
</AccordionGroup>
