# Стакан (DOM)

Виджет DOM отображает книгу ордеров в реальном времени в виде ценовой лестницы с объёмами bid и ask на каждом ценовом уровне. Он обеспечивает прямую видимость рыночной ликвидности, крупных отложенных ордеров и потенциальной айсберг-активности.

{/* screenshot: DOM widget showing price ladder with bids and asks */}

## Обзор

DOM показывает:

- **Столбец цены** — центральный столбец с ценовыми уровнями
- **Объём bid** — левая сторона, показывающая ордера на покупку на каждом уровне цены
- **Объём ask** — правая сторона, показывающая ордера на продажу на каждом уровне цены
- **Спред** — подсвеченный разрыв между лучшим bid и лучшим ask
- **Последняя сделка** — отмечена на ценовой лестнице маркером

Все данные передаются в реальном времени через WebSocket, обновляясь несколько раз в секунду.

## Открытие DOM

<Steps>
  <Step title="Добавьте панель DOM">
    Щёлкните правой кнопкой мыши по заголовку вкладки и выберите **Добавить DOM** из контекстного меню, или используйте селектор типа графика для переключения панели в режим DOM.
  </Step>
  <Step title="Выберите символ">
    DOM использует тот же символ, что и активная вкладка графика. Измените символ в панели инструментов для обновления DOM.
  </Step>
</Steps>

## Режимы отображения

### Стандартный режим

Вид по умолчанию показывает сырую глубину книги ордеров с числами объёмов на каждом ценовом уровне. Полосы расширяются горизонтально пропорционально размеру объёма относительно видимого диапазона.

### Градиентный режим

Градиентный режим заменяет числовые полосы объёмов плавным цветовым градиентом, что упрощает обнаружение крупных концентраций ликвидности с первого взгляда.

| Визуал | Значение |
|--------|----------|
| **Насыщенный цвет** | Большой объём на этом ценовом уровне |
| **Бледный цвет** | Малый объём |
| **Зелёный градиент** | Сторона bid |
| **Красный градиент** | Сторона ask |

<Tip>
  Градиентный режим особенно полезен на дисплеях с высоким разрешением, где он превращает весь DOM в тепловую визуализацию структуры ликвидности.
</Tip>

## Обнаружение айсберг-ордеров

DOM включает алгоритм обнаружения айсберг-ордеров. Айсберг-ордера — это крупные ордера, разделённые на меньшие видимые части: когда одна часть исполняется, появляется другая на той же цене.

### Как обнаруживаются айсберги

Система отслеживает повторное пополнение на одном и том же ценовом уровне:

1. Видимый ордер на ценовом уровне исполняется (частично или полностью).
2. В течение короткого временного окна на той же цене появляется новый ордер.
3. Если этот паттерн повторяется выше настраиваемого порога, уровень помечается как айсберг.

Обнаруженные айсберги отмечаются характерным визуальным индикатором на ценовой лестнице.

{/* screenshot: DOM with iceberg detection highlight */}

<Note>
  Обнаружение айсбергов работает только на живых данных. Оно не восстанавливает исторические айсберги из прошлых снимков книги ордеров.
</Note>

## Фильтры объёмов

Фильтры объёмов позволяют подсвечивать или скрывать ценовые уровни на основе размера ордеров. Это помогает сосредоточиться на значимых уровнях ликвидности, игнорируя шум.

### Размерные категории

Настройте пороги размеров для категоризации ордеров:

| Категория | Описание | Визуал |
|-----------|----------|--------|
| **Малые** | Ниже первого порога | Обычное отображение |
| **Средние** | Между первым и вторым порогом | Слегка подсвечены |
| **Крупные** | Между вторым и третьим порогом | Заметно подсвечены |
| **Киты** | Выше третьего порога | Яркая подсветка с отдельным цветом |

### Настройка фильтра

<Steps>
  <Step title="Откройте настройки фильтра">
    Нажмите значок фильтра на панели инструментов DOM, чтобы открыть панель фильтра объёмов.
  </Step>
  <Step title="Установите пороги">
    Введите пороговые значения объёмов для каждой размерной категории. Значения указываются в котируемой валюте (USDT для USDT-M фьючерсов).
  </Step>
  <Step title="Выберите режим отображения">
    Выберите, **подсвечивать** крупные ордера (по умолчанию) или **скрывать** мелкие ордера ниже минимального порога.
  </Step>
</Steps>

### Именованные пресеты

Сохраняйте конфигурации фильтров как именованные пресеты для быстрого переключения между различными рыночными условиями.

- **Скальпинг** — низкие пороги для отображения мелких значимых ордеров на 1m графиках
- **Свинг** — высокие пороги для фокуса только на основных уровнях ликвидности
- **Наблюдение за китами** — очень высокие пороги для показа только крупнейших отложенных ордеров

<Tip>
  Создавайте разные пресеты фильтров для разных символов. «Крупный» ордер на BTCUSDT сильно отличается от «крупного» ордера на низколиквидном альткоине.
</Tip>

## DOM на графике

Помимо отдельного виджета DOM, вы можете наложить уровни книги ордеров прямо на основной график с помощью индикатора **DOM on Chart**. Он отображает глубину bid/ask в реальном времени в виде горизонтальных полос на свечном или кластерном графике.

Включите его из панели индикаторов в категории **Поток ордеров**.

## Замечания по производительности

DOM обрабатывает большой объём WebSocket-сообщений. На медленных соединениях или оборудовании:

- Уменьшите количество видимых ценовых уровней (увеличьте масштаб до более узкого диапазона)
- Отключите градиентный режим, если рендеринг замедлен
- Закрывайте неиспользуемые панели DOM — каждая поддерживает собственную WebSocket-подписку

<Warning>
  Запуск нескольких панелей DOM для разных символов одновременно значительно увеличивает сетевой трафик и нагрузку на процессор. Используйте одну панель DOM за раз для лучшей производительности.
</Warning>
