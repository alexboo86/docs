Cluster Terminal — это профессиональный терминал для торговли криптовалютными
фьючерсами, созданный для кластерного анализа, объёмного профилирования и
исследования потока ордеров. Это руководство проведёт вас через полный процесс
установки — от проверки системных требований до первого запуска приложения.

## Системные требования

Перед загрузкой убедитесь, что ваш компьютер соответствует минимальным
характеристикам. Cluster Terminal использует WebGL 2.0 для высокопроизводительного
рендеринга графиков и поддерживает постоянное WebSocket-соединение для получения
рыночных данных в реальном времени.

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| **Операционная система** | Windows 10 64-bit (версия 1809+) | Windows 11 64-bit |
| **Оперативная память** | 4 ГБ | 8 ГБ и более |
| **Процессор** | Двухъядерный, 2 ГГц | Четырёхъядерный, 3 ГГц+ |
| **Видеокарта** | Совместимая с WebGL 2.0 (встроенная) | Дискретная GPU (NVIDIA / AMD) |
| **Место на диске** | 200 МБ для приложения | 500 МБ включая кэш и данные |
| **Экран** | 1366 x 768 | 1920 x 1080 или выше |
| **Сеть** | Стабильное широкополосное соединение | Соединение с низкой задержкой (< 50 мс до сервера) |

<Note>
  Cluster Terminal отрисовывает все графические поверхности — свечи, кластеры, тепловые карты и оверлеи — с использованием WebGL 2.0 через движок SciChart. Если драйверы вашей видеокарты устарели, приложение может переключиться на программный рендеринг, что значительно снижает производительность. Обновите драйверы видеокарты до последней версии перед установкой.
</Note>

### Проверка поддержки WebGL

Если вы не уверены, поддерживает ли ваша система WebGL 2.0, откройте любой
браузер на основе Chromium (Chrome, Edge) и перейдите по адресу `chrome://gpu`.
Найдите строку **WebGL2: Hardware accelerated**. Если указано «Software only»
или «Unavailable», необходимо обновить драйверы видеокарты или проверить, что
аппаратное ускорение включено в BIOS вашей системы.

<Tip>
  Cluster Terminal поставляется со встроенной средой Chromium (через Tauri),
  поэтому настройки WebGL на уровне браузера не применяются напрямую. Однако те
  же требования к драйверам GPU актуальны. Если WebGL работает в Chrome, он
  будет работать и в Cluster Terminal.
</Tip>

## Загрузка Cluster Terminal

Cluster Terminal распространяется как стандартный установщик Windows NSIS.
Последнюю версию можно получить одним из следующих способов:

- **Официальный канал распространения** — загрузите по ссылке, предоставленной
  вашим администратором или из официальной группы Cluster Terminal в Telegram.
- **Прямая ссылка от администратора сервера** — если ваша организация использует
  собственный сервер, администратор может предоставить прямую ссылку для
  скачивания из панели администрирования.

Файл установщика имеет следующее имя:

```
ClusterTerminal_X.Y.Z_x64-setup.exe
```

где `X.Y.Z` — номер версии (например, `1.2.0`).

<Warning>
  Загружайте установщик только из официальных или доверенных источников.
  Не используйте сторонние сайты загрузки. Бинарный файл установщика подписан
  ключом Ed25519, который автообновление проверяет при каждом последующем
  обновлении.
</Warning>

## Запуск установщика

<Steps>
  <Step title="Запустите установщик">
    Дважды щёлкните `ClusterTerminal_X.Y.Z_x64-setup.exe`, чтобы запустить
    мастер установки NSIS. Если Windows SmartScreen отображает предупреждение,
    поскольку приложение ещё не получило широкого распространения, нажмите
    **Подробнее**, а затем **Выполнить в любом случае**.

    {/* screenshot: Windows SmartScreen warning with "More info" and "Run anyway" highlighted */}
  </Step>

  <Step title="Примите лицензионное соглашение">
    Ознакомьтесь с лицензионным соглашением конечного пользователя и нажмите
    **Принимаю** для продолжения.
  </Step>

  <Step title="Выберите каталог установки">
    Путь по умолчанию — `C:\Program Files\Cluster Terminal`. Вы можете изменить
    его на любой каталог, но рекомендуется размещение на SSD для более быстрого
    запуска и операций с кэшем. Нажмите **Далее** для продолжения.

    {/* screenshot: NSIS directory selection dialog */}
  </Step>

  <Step title="Выберите папку в меню «Пуск»">
    Выберите или подтвердите название папки в меню «Пуск». По умолчанию —
    **Cluster Terminal**. Нажмите **Установить**, чтобы начать установку.
  </Step>

  <Step title="Дождитесь завершения установки">
    Установщик копирует файлы приложения, регистрирует автообновление и
    создаёт ярлыки на рабочем столе и в меню «Пуск». Обычно это занимает
    менее 30 секунд.

    {/* screenshot: NSIS installation progress bar */}
  </Step>

  <Step title="Завершение и запуск">
    Когда индикатор прогресса заполнится, нажмите **Готово**. Если установлен
    флажок «Запустить Cluster Terminal», приложение запустится немедленно.
  </Step>
</Steps>

<Tip>
  Установщик не требует перезагрузки системы. Вы можете начать использовать
  Cluster Terminal сразу после завершения установки.
</Tip>

## Первый запуск — подключение к серверу данных

При самом первом запуске Cluster Terminal необходимо указать, откуда загружать
рыночные данные. Это сервер данных, предоставляющий исторические свечи,
кластерные объёмы, открытый интерес и потоковый WebSocket-канал в реальном
времени.

<Steps>
  <Step title="Введите URL сервера">
    Появится экран настройки с запросом **URL сервера**. Это адрес сервера
    данных Cluster Terminal. Если вы получили URL от администратора (например,
    `https://data.example.com`), введите его здесь. При использовании
    размещённого сервиса по умолчанию URL может быть уже предзаполнен.

    {/* screenshot: Server URL input on first launch */}
  </Step>

  <Step title="Проверьте подключение">
    После ввода URL приложение попытается подключиться. Зелёный индикатор
    подтверждает успешное подключение. Если индикатор стал красным,
    перепроверьте URL и сетевое подключение.
  </Step>

  <Step title="Начните бесплатный пробный период">
    После подключения автоматически начинается **7-дневный бесплатный пробный
    период**. Не требуется создание аккаунта, адрес электронной почты или
    банковская карта. Пробный период даёт доступ ко всем основным функциям
    для оценки платформы.

    {/* screenshot: Trial started confirmation banner */}
  </Step>
</Steps>

<Note>
  URL сервера сохраняется локально на вашем устройстве и не нуждается в
  повторном вводе, если вы не переустановите приложение или не переключитесь
  на другой сервер. Изменить его можно позже в **Настройках**.
</Note>

### Что предоставляет сервер

Сервер данных является единственным источником всей рыночной информации,
отображаемой в Cluster Terminal:

| Тип данных | Описание |
|-----------|----------|
| **Исторические свечи** | Свечные данные OHLCV по всем поддерживаемым таймфреймам |
| **Кластерные объёмы** | Разбивка объёма покупок/продаж по ценовым уровням для каждой свечи |
| **Открытый интерес** | Исторические данные и данные открытого интереса в реальном времени |
| **Снимки стакана** | Данные глубины рынка для виджета DOM |
| **Потоковый канал** | WebSocket-поток для обновлений свечей, сделок и объёмов в реальном времени |

Все данные передаются через протоколы HTTPS (REST) и WebSocket (WSS).
Сторонние API или плагины браузера не требуются.

## Система автообновления

Cluster Terminal включает встроенный механизм автообновления, поддерживающий
приложение в актуальном состоянии без ручного вмешательства.

### Как это работает

1. **Автоматическая проверка** — при каждом запуске приложение запрашивает
   сервер обновлений на наличие новых релизов.
2. **Баннер уведомления** — если доступна новая версия, в верхней части окна
   появляется баннер с номером версии и описанием изменений.
3. **Загрузка одним нажатием** — нажмите кнопку **Обновить** в баннере. Новый
   установщик загружается в фоновом режиме, пока вы продолжаете работать.
4. **Установка при перезапуске** — после завершения загрузки нажмите
   **Перезапустить сейчас** для применения обновления. Приложение закрывается,
   установщик NSIS запускается в тихом режиме, и новая версия запускается
   автоматически.

{/* screenshot: Update notification banner with version number and "Update" button */}

<Warning>
  Не закрывайте приложение принудительно во время загрузки или установки
  обновления. Дождитесь завершения перезапуска, чтобы избежать повреждения
  установки.
</Warning>

### Безопасность обновлений

Каждый бинарный файл обновления подписан ключом Ed25519. Приложение
проверяет подпись перед применением любого обновления. Если подпись не
совпадает, обновление отклоняется и отображается ошибка. Это гарантирует,
что на ваш компьютер устанавливаются только подлинные, не изменённые релизы.

<Note>
  Обновления являются инкрементальными. Вам не нужно повторно загружать полный
  установщик. Система обновления загружает только установочный файл новой версии
  и применяет его автоматически.
</Note>

## Устранение проблем при установке

### Предупреждение Windows SmartScreen

Поскольку Cluster Terminal является специализированным приложением и может
ещё не иметь широкого распространения, Windows SmartScreen может пометить
установщик как «неопознанный». Это стандартное предупреждение для новых
приложений и не указывает на угрозу безопасности.

**Решение:** нажмите **Подробнее** в диалоге SmartScreen, затем
**Выполнить в любом случае**.

### Ложные срабатывания антивируса

Некоторые антивирусные программы могут поместить в карантин или заблокировать
установщик или исполняемый файл приложения. Это может произойти из-за того,
что приложение использует WebSocket-соединения и паттерны доступа к файловой
системе, которые эвристические сканеры иногда отмечают.

**Решение:**
- Добавьте исключение для каталога установки Cluster Terminal
  (по умолчанию: `C:\Program Files\Cluster Terminal`).
- Добавьте исключение для каталога данных приложения:
  `%APPDATA%\com.clusterterminal.app`.
- Если антивирус поместил установщик в карантин, восстановите его и добавьте
  в список разрешённых перед повторным запуском.

<Tip>
  Если вы находитесь в корпоративной среде с управляемым антивирусом, попросите
  IT-администратора добавить в белый список исполняемый файл Cluster Terminal
  и путь установки.
</Tip>

### Проблемы с файрволом и сетью

Cluster Terminal требует исходящего доступа по следующим протоколам:

| Протокол | Порт | Назначение |
|----------|------|-----------|
| HTTPS | 443 | REST API для исторических данных, проверки лицензии, обновлений |
| WSS | 443 (или пользовательский) | WebSocket для рыночных данных в реальном времени |

Если вы находитесь за корпоративным файрволом или прокси:

- Убедитесь, что исходящие HTTPS и WebSocket-соединения разрешены к домену
  или IP-адресу вашего сервера.
- Если ваша организация использует прокси, Cluster Terminal учитывает
  системные настройки прокси, настроенные в Windows.
- VPN-соединения могут увеличить задержку WebSocket-канала, что может
  вызвать задержки обновлений в реальном времени.

### Проблемы рендеринга WebGL

Если графики отображаются пустыми, мерцают или рендерятся некорректно:

1. **Обновите драйверы GPU** — посетите сайт производителя видеокарты (NVIDIA,
   AMD или Intel) и установите последние драйверы.
2. **Отключите аппаратное ускорение в других приложениях** — ресурсоёмкие
   приложения, конкурирующие за доступ к GPU, могут вызывать проблемы рендеринга.
   Закройте ненужные GPU-интенсивные программы.
3. **Проверьте встроенную vs. дискретную GPU** — на ноутбуках с двумя
   видеокартами убедитесь, что Cluster Terminal настроен на использование
   дискретной GPU. Настроить это можно в Панели управления NVIDIA или
   AMD Radeon Settings в разделе назначения GPU для приложений.

### Расположение данных приложения

Cluster Terminal хранит конфигурацию, данные лицензии и кэшированные файлы в:

```
%APPDATA%\com.clusterterminal.app
```

Этот каталог содержит:

- **Лицензия и состояние пробного периода** — локально кэшированные данные проверки.
- **URL сервера** — настроенный адрес сервера данных.
- **Данные рабочих пространств** — локальные копии рабочих пространств (макеты,
  индикаторы, рисунки).
- **Журналы приложения** — диагностические журналы, полезные для устранения неполадок.

<Warning>
  Удаление этого каталога сбрасывает приложение в начальное состояние, включая
  состояние пробного периода и все локально сохранённые рабочие пространства.
  Сделайте резервную копию рабочих пространств перед очисткой данных приложения.
</Warning>

### Переустановка

Если вам необходимо выполнить чистую переустановку:

1. Удалите Cluster Terminal через **Параметры Windows > Приложения > Установленные приложения**.
2. При необходимости удалите каталог данных приложения
   `%APPDATA%\com.clusterterminal.app` для полного сброса.
3. Загрузите и запустите последний установщик.

<Note>
  Переустановка на том же устройстве сохраняет отпечаток устройства. Если у вас
  была активирована лицензия, она повторно использует тот же слот устройства и
  не будет учтена как новое устройство.
</Note>

[Первые шаги](/getting-started/first-steps), чтобы узнать, как настроить
графики, добавить индикаторы и сохранить макет.
