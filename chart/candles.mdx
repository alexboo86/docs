Свечные графики — основа технического анализа. В Cluster Terminal свечной режим отображает стандартные японские свечи на основе OHLC-данных Binance Futures & Spot с полным контролем над цветами, границами, тенями, прозрачностью тела, скруглением углов и эффектами свечения.

## Что показывают свечные графики

Каждая свеча представляет ценовое движение за один временной период (определяемый выбранным таймфреймом). Свеча содержит четыре значения:

| Компонент | Описание |
|-----------|----------|
| **Open (Открытие)** | Цена в начале периода |
| **High (Максимум)** | Наивысшая цена за период |
| **Low (Минимум)** | Наименьшая цена за период |
| **Close (Закрытие)** | Цена в конце периода |

Прямоугольное **тело** свечи охватывает диапазон от цены открытия до закрытия. Тонкие линии, выходящие вверх и вниз от тела, называются **тенями** (или фитилями) и отражают максимум и минимум периода.

- **Бычья свеча** (закрытие > открытие) указывает на рост цены за период. По умолчанию бычьи свечи окрашены в зелёный цвет.
- **Медвежья свеча** (закрытие < открытие) указывает на снижение цены за период. По умолчанию медвежьи свечи окрашены в красный цвет.

Когда цены открытия и закрытия равны (или почти равны), свеча выглядит как тонкая горизонтальная линия, называемая **доджи**, сигнализирующая о неопределённости между покупателями и продавцами.

## Выбор таймфрейма

Таймфрейм определяет длительность, которую представляет каждая свеча. Cluster Terminal поддерживает следующие таймфреймы:

| Таймфрейм | Длительность свечи | Типичное применение |
|-----------|-------------------|---------------------|
| **1m** | 1 минута | Скальпинг, точные входы |
| **5m** | 5 минут | Краткосрочная внутридневная торговля |
| **15m** | 15 минут | Внутридневные свинг-сетапы |
| **30m** | 30 минут | Анализ на уровне сессии |
| **1h** | 1 час | Определение внутридневного тренда |
| **4h** | 4 часа | Свинг-трейдинг, межсессионные тренды |
| **1d** | 1 день | Дневной уклон, макроструктура |
| **1w** | 1 неделя | Анализ долгосрочного тренда |

Выберите таймфрейм из выпадающего списка в панели инструментов графика. Смена таймфрейма перезагружает свечные данные с сервера и перерисовывает график.

<Note>
Все свечные данные формируются из агрегированных сделок Binance Futures & Spot и вычисляются на стороне сервера. Сервер хранит необработанные тиковые данные и динамически строит свечи для запрошенного таймфрейма, обеспечивая точность на всех интервалах.
</Note>

## Цвета тела

Cluster Terminal использует раздельные настройки цвета для тел бычьих и медвежьих свечей. Настроить их можно в **Внешний вид > Переопределение цветов** или через панель настроек графика.

### Цвет тела бычьей (растущей) свечи

Цвет заливки для свечей, где закрытие больше или равно открытию. По умолчанию используется зелёный оттенок, оптимизированный для тёмных тем.

### Цвет тела медвежьей (падающей) свечи

Цвет заливки для свечей, где закрытие меньше открытия. По умолчанию используется красный оттенок, обеспечивающий чёткий контраст с бычьим цветом на любой теме.

<Tip>
Выбирайте цвета с достаточным контрастом между бычьими и медвежьими свечами. Избегайте слишком похожих по оттенку или яркости цветов, так как это затрудняет чтение графика. Многие профессиональные трейдеры используют пары зелёный/красный или синий/оранжевый.
</Tip>

### Прозрачность тела

Настройка **bodyOpacity** управляет прозрачностью заливки тела свечи. Значение по умолчанию — **0.88** (88% непрозрачности), что обеспечивает слегка полупрозрачный вид. Уменьшите прозрачность, если хотите, чтобы индикаторы или оверлеи за свечами были видны.

| Диапазон прозрачности | Визуальный эффект |
|----------------------|------------------|
| **1.0** | Полностью непрозрачные, сплошные свечи |
| **0.7 -- 0.9** | Слегка полупрозрачные, стандартный вид |
| **0.4 -- 0.6** | Полупрозрачные, оверлеи видны под ними |
| **Ниже 0.3** | Призрачные свечи, преимущественно для фонового контекста |

## Границы и контуры

Каждое тело свечи может отображать отдельную границу (контур) с независимым управлением цветом.

### Толщина границы

Настройка **borderWidth** управляет толщиной контура тела свечи в пикселях. По умолчанию — **1px**. Установите **0**, чтобы полностью отключить границы и показывать только залитые тела.

### Цвета границ

Цвета границ настраиваются отдельно для бычьих и медвежьих свечей:

- **Граница растущей свечи** (`candleUpBorder`) -- цвет контура бычьих свечей. Если не задан, используется чуть более тёмный оттенок цвета тела.
- **Граница падающей свечи** (`candleDownBorder`) -- цвет контура медвежьих свечей. Если не задан, используется чуть более тёмный оттенок цвета тела.

<Note>
Независимая настройка цветов границ от цветов тела полезна для создания эффекта «пустых свечей» — например, прозрачное тело с видимым цветным контуром или залитое тело с контрастной границей для акцента.
</Note>

## Тени (фитили)

Тени простираются вертикально от тела свечи до максимальной и минимальной цен. Их цвет настраивается независимо от тела и границы.

### Цвета теней

- **Тень растущей свечи** (`candleUpShadow`) -- цвет фитиля бычьих свечей.
- **Тень падающей свечи** (`candleDownShadow`) -- цвет фитиля медвежьих свечей.

По умолчанию цвета теней совпадают с цветами тела. Некоторые трейдеры предпочитают приглушённые или серые фитили, чтобы сосредоточить внимание на теле. Другие устанавливают единый нейтральный цвет для всех фитилей независимо от направления свечи.

## Скругление углов

Настройка **cornerRadius** скругляет углы тела свечи. По умолчанию — **1** (минимальное скругление). Увеличьте значение для более мягкого, современного вида. Установите **0** для идеально острых прямоугольных свечей.

| Значение | Вид |
|----------|-----|
| **0** | Острые прямоугольные углы |
| **1** | Незначительное скругление (по умолчанию) |
| **2 -- 4** | Заметное скругление, капсулоподобный вид |

## Эффект свечения

Настройка **glowIntensity** добавляет светящийся ореол вокруг тел свечей. Значение по умолчанию — **0.28** (слабое свечение). Эффект использует цвет тела для создания мягкого ореола, выделяющего свечи на фоне.

| Значение | Эффект |
|----------|--------|
| **0** | Без свечения |
| **0.1 -- 0.3** | Слабое фоновое свечение (диапазон по умолчанию) |
| **0.4 -- 0.7** | Умеренное свечение, свечи «выступают» из фона |
| **0.8 -- 1.0** | Интенсивное неоновое свечение, высокий визуальный эффект |

<Tip>
Эффекты свечения лучше всего работают на тёмных темах. На светлых темах уменьшите интенсивность или отключите полностью, так как свечение может снижать контраст и затруднять чтение свечей.
</Tip>

## Настройка внешнего вида свечей

<Steps>
  <Step title="Откройте панель настроек графика">
    Нажмите **значок шестерёнки** в панели инструментов графика или щёлкните правой кнопкой по фону графика и выберите **Настройки**.
  </Step>
  <Step title="Выберите пресет «Свечи»">
    В выпадающем списке пресетов в верхней части панели настроек выберите **Свечи**. Это активирует свечной режим отображения без кластерных оверлеев.
  </Step>
  <Step title="Настройте стиль свечей">
    В разделе **Стиль свечей** настройте скругление углов, прозрачность тела, толщину границы и интенсивность свечения.
  </Step>
  <Step title="Переопределите цвета">
    Переключитесь на панель **Внешний вид** и измените переопределения цветов для тела, границы и тени бычьих/медвежьих свечей. Здесь же можно сменить тему графика.
  </Step>
</Steps>

## Когда использовать свечной режим

Свечной режим — самый лёгкий режим рендеринга в Cluster Terminal. Он не загружает и не отрисовывает данные кластерных объёмов, что делает его идеальным для нескольких сценариев:

### Обзор на старшем таймфрейме

Используйте свечной режим на графиках **4h** или **1d** для определения общего тренда и ключевой ценовой структуры. Кластерные данные менее полезны на этих таймфреймах, поскольку объём агрегирован за очень длительные периоды.

### Многографиковые макеты

При одновременной работе с **четырьмя и более графиками** переключение некоторых панелей в свечной режим значительно снижает нагрузку на GPU и память. Каждый кластерный график требует рендеринга сотен или тысяч отдельных ячеек; свечной график отрисовывает только тела и фитили свечей.

### Анализ на основе паттернов

Если ваша торговая методология сосредоточена на свечных паттернах (поглощение, доджи, молот, пин-бар) или классических графических паттернах (голова и плечи, треугольники, клинья), свечной режим обеспечивает чистый, незагромождённый вид.

<Tip>
Распространённая профессиональная настройка — использование **свечей на старшем таймфрейме** (1h или 4h) для определения направления и **кластеров на младшем таймфрейме** (1m или 5m) для точного тайминга входа. Это сочетает ясность свечного анализа с глубиной данных потока ордеров.
</Tip>

## Свечной режим vs. другие режимы

| Функция | Свечи | Кластеры | Тепловая карта |
|---------|-------|----------|---------------|
| Ценовое движение (OHLC) | Полное | Полное | С оверлеем свечей |
| Объём по ценовому уровню | Нет | Да | Нет |
| Разбивка Bid/Ask | Нет | Да | Нет |
| Глубина стакана | Нет | Нет | Да |
| Нагрузка на рендеринг | Низкая | Высокая | Средняя |
| Лучшие таймфреймы | Все | 1m -- 1h | 1m -- 15m |

## Справочник переопределения цветов

Все переопределения цветов свечей находятся в настройках **Внешний вид**. Они переопределяют значения по умолчанию активной темы:

| Ключ переопределения | Управляет | По умолчанию |
|---------------------|-----------|-------------|
| `candleUpBg` | Заливка тела бычьей свечи | Зелёный темы |
| `candleDownBg` | Заливка тела медвежьей свечи | Красный темы |
| `candleUpBorder` | Контур тела бычьей свечи | Производный от тела |
| `candleDownBorder` | Контур тела медвежьей свечи | Производный от тела |
| `candleUpShadow` | Цвет фитиля бычьей свечи | Совпадает с телом |
| `candleDownShadow` | Цвет фитиля медвежьей свечи | Совпадает с телом |

<Note>
Переопределения цветов сохраняются независимо от выбранной темы. Смена темы не сбрасывает переопределения. Используйте кнопку **Сбросить к значениям темы** в панели «Внешний вид» для очистки всех настроек.
</Note>

## Горячие клавиши

| Сочетание | Действие |
|-----------|---------|
| Колесо прокрутки | Масштабирование по оси времени |
| Нажатие + перетаскивание | Перемещение графика |
| Двойное нажатие | Сброс масштаба к диапазону по умолчанию |
| `Esc` | Снять выделение со всех рисунков / закрыть панели |
